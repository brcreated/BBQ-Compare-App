# BBQ Comparison App — Showroom UX Rules

## Goal

The BBQ Comparison App is designed for showroom environments.

It must feel like a **premium interactive product display**, not a typical web application.

Primary devices:

- iPad (landscape)
- Android touchscreen displays
- desktop browsers

---

## Layout Principles

The UI must prioritize:

- large touch targets
- clear visual hierarchy
- easy scanning from standing distance
- landscape orientation

The comparison rail should remain visible when possible.

---

## Interaction Principles

The interface should prioritize:

- taps over typing
- large buttons
- minimal scrolling
- visual exploration

Avoid:

- small links
- dense tables
- hover-only interactions

---

## Visual Design Principles

The experience should feel:

- premium
- clean
- image-focused
- modern

Use:

- large product imagery
- clear brand visuals
- smooth animations
- spacious layouts

Avoid:

- cluttered layouts
- excessive text
- overly complex navigation

---

## Performance Rules

The showroom experience must:

- load extremely quickly
- avoid blocking screens
- preload important assets
- lazy-load heavy content

Animations should remain smooth on touchscreen devices.

---

## Product Exploration Flow

The primary flow should support:

Browse  
→ Select product  
→ Add to comparison  
→ Compare side-by-side  
→ Explore product details

The comparison experience is the centerpiece of the application.
